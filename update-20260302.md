# Image Picker Migration Guide

## Bối cảnh vấn đề

App dùng **Scene-based lifecycle (UIWindowScene)** trên iOS 13+. `react-native-image-crop-picker` v0.51.1 (latest) vẫn dùng deprecated API:

```objc
// ❌ Lỗi trên Scene-based app — [AppDelegate window] trả về nil
UIViewController *root = [[[[UIApplication sharedApplication] delegate] window] rootViewController];
```

Hậu quả: bấm "Chọn từ thư viện" → popup tắt, không mở photo library. Bấm "Chụp ảnh" → crash hoặc không hiện camera.

---

## Giải pháp tổng thể

1. **Patch native iOS** của `react-native-image-crop-picker` bằng `patch-package` để dùng `connectedScenes` API
2. **Bỏ `react-native-image-picker`** (đã cài nhưng không cần thiết sau khi patch crop-picker)
3. **Dùng `react-native-image-crop-picker` trực tiếp** cho tất cả các màn hình

---

```typescript
const requestCameraPermission = async (): Promise<boolean> => {
  if (Platform.OS === 'android') {
    const granted = await PermissionsAndroid.request(
      PermissionsAndroid.PERMISSIONS.CAMERA,
      {
        title: 'Quyền truy cập Camera',
        message: 'Ứng dụng cần quyền truy cập camera để chụp ảnh.',
        buttonNeutral: 'Hỏi sau',
        buttonNegative: 'Từ chối',
        buttonPositive: 'Đồng ý',
      }
    );
    return granted === PermissionsAndroid.RESULTS.GRANTED;
  }
  return true;
};

const handleTakePhoto = async () => {
  try {
    if (!(await requestCameraPermission())) {
      Alert.alert('Lỗi', 'Bạn cần cấp quyền truy cập camera để tiếp tục.');
      return;

    if (error.code !== 'E_PICKER_CANCELLED') {
      Alert.alert('Lỗi', 'Không thể chụp ảnh. Vui lòng thử lại.');
    }
  }
};

const handlePickFromLibrary = async () => {
  try {
    const image = await ImagePicker.openPicker({
      multiple: false,
      mediaType: 'photo',
      compressImageQuality: 0.8,
    });
    setImage({ src: image.path, uri: image.path });
  } catch (error: any) {
    if (error.code !== 'E_PICKER_CANCELLED') {
      Alert.alert('Lỗi', 'Không thể chọn ảnh. Vui lòng thử lại.');
    }
  }
};
```

### 3.4 Màn hình báo cáo — Chọn nhiều ảnh

```typescript
const [images, setImages] = useState<string[]>([]);

const handleTakePhoto = async () => {
  try {
    if (Platform.OS === 'android') {
      // ... request camera permission (xem 3.2)
    }
    const image = await ImagePicker.openCamera({
      mediaType: 'photo',
      compressImageQuality: 0.8,
    });
    setImages(prev => [...prev, image.path]);
  } catch (error: any) {
    if (error.code !== 'E_PICKER_CANCELLED') {
      Alert.alert('Lỗi', 'Không thể chụp ảnh. Vui lòng thử lại.');
    }
  }
};

const handlePickFromLibrary = async () => {
  try {
    const selectedImages = await ImagePicker.openPicker({
      multiple: true,
      mediaType: 'photo',
      compressImageQuality: 0.8,
    });
    const paths = selectedImages.map(img => img.path);
    setImages(prev => [...prev, ...paths]);
  } catch (error: any) {
    if (error.code !== 'E_PICKER_CANCELLED') {
      Alert.alert('Lỗi', 'Không thể chọn ảnh. Vui lòng thử lại.');
    }
  }
};

const handleRemoveImage = (index: number) => {
  setImages(prev => prev.filter((_, i) => i !== index));
};
```

## Bug Fix — Avatar không cập nhật ngay sau khi upload (ProfileScreen)

### Triệu chứng

Upload avatar thành công (có log URL đúng), nhưng avatar vẫn hiển thị ảnh cũ. Phải chuyển sang screen khác rồi quay lại mới thấy ảnh mới.

### Root Cause

URL ảnh mới khác URL cũ (server tạo filename random mỗi lần). Tuy nhiên React Native `<Image>` không tải ngay vì có độ trễ giữa lúc `uri` prop thay đổi và lúc image thực sự load từ network — khoảng thời gian đó bị `Alert.alert()` hiển thị ngay sau `updateAvatar()` gây interrupt, làm ảnh không load được. Khi navigate đi rồi về, component mount lại fresh nên load bình thường.

Các cách fix sai đã thử:
- Thêm `?t=timestamp` vào URL → server/CDN không hỗ trợ query string → `onError` fire → show default avatar
- Dùng `avatarKey` + `useEffect` để force remount Avatar → vẫn cùng timing issue với network load

### Fix đúng

Dùng **local file path** hiển thị ngay lập tức trong lúc upload. Khi upload xong mới switch sang server URL.

**`ProfileScreen.tsx`:**

```typescript
// Thêm state
const [localAvatarUri, setLocalAvatarUri] = useState<string | undefined>(undefined);

// Trong uploadImageToServer:
const uploadImageToServer = async (imagePath: string) => {
  try {
    setIsUploading(true);
    // Hiển thị ảnh local ngay lập tức, không cần chờ network
    setLocalAvatarUri(imagePath);

    const response = await uploadService.uploadAvatar(imagePath);
    const avatarUrl = response.data || imagePath;
    updateAvatar(avatarUrl);       // Lưu server URL vào Zustand + storage
    setLocalAvatarUri(undefined);  // Clear local override, dùng server URL

    Alert.alert('Thành công', 'Ảnh đại diện đã được cập nhật!');
  } catch (error: any) {
    setLocalAvatarUri(undefined);
    Alert.alert('Lỗi', error.message || 'Không thể upload ảnh. Vui lòng thử lại.');
  } finally {
    setIsUploading(false);
  }
};

// Trong JSX — ưu tiên local path, fallback về store:
<Avatar uri={localAvatarUri || user?.avatar} size={100} />
```

**Bỏ các thứ không cần thiết:**
- Xóa `avatarKey` state và `useEffect([user?.avatar])` → không cần force remount
- Xóa `useEffect` import nếu không dùng ở nơi khác

### Nguyên lý

| Giai đoạn | `localAvatarUri` | `user?.avatar` | Avatar hiển thị |
|-----------|-----------------|----------------|-----------------|
| Trước upload | `undefined` | URL cũ | ảnh cũ |
| Đang upload | `file:///...` | URL cũ | ảnh mới (local, tức thì) |
| Upload xong | `undefined` | URL mới | ảnh mới (server URL) |
| Lỗi upload | `undefined` | URL cũ | ảnh cũ (revert) |

Local file path load từ disk → **không cần network → không bị interrupt** → hiển thị ngay lập tức.

### Checklist khi áp dụng cho project mới

- [ ] Thêm `localAvatarUri` state vào ProfileScreen
- [ ] Set `localAvatarUri = imagePath` trước khi gọi upload API
- [ ] Clear `localAvatarUri` sau khi upload xong (cả success lẫn error)
- [ ] Avatar render dùng `localAvatarUri || user?.avatar`
- [ ] Xóa `avatarKey` + `useEffect([user?.avatar])` nếu có

## Fix Firebase Messaging Warnings (React Native Firebase v22+)

### Triệu chứng

Rất nhiều warning dạng:

```
This method is deprecated (as well as all React Native Firebase namespaced API)
and will be removed in the next major release as part of move to match Firebase
Web modular SDK API. Please see migration guide for more details:
https://rnfirebase.io/migrating-to-v22 Please use `getApp()` instead.
```

### Root Cause

Từ `@react-native-firebase` v22+, toàn bộ **namespaced API** (`messaging().method()`) bị deprecated. Phải dùng **modular API** (import từng function riêng lẻ).

### Fix — Rewrite `notificationService.ts`

Thay toàn bộ import và cách gọi:

```typescript
// ❌ Cũ — namespaced API (deprecated v22+)
import messaging from '@react-native-firebase/messaging';

messaging().setBackgroundMessageHandler(handler);
const authStatus = await messaging().requestPermission();
messaging().isDeviceRegisteredForRemoteMessages
await messaging().registerDeviceForRemoteMessages();
const token = await messaging().getToken();
messaging().onMessage(handler);
messaging().onNotificationOpenedApp(handler);
messaging().getInitialNotification();
await messaging().subscribeToTopic(topic);
await messaging().unsubscribeFromTopic(topic);
authStatus === messaging.AuthorizationStatus.AUTHORIZED
```

```typescript
// ✅ Mới — modular API
import {
  getMessaging,
  getToken,
  onMessage,
  onNotificationOpenedApp,
  getInitialNotification,
  isDeviceRegisteredForRemoteMessages,
  registerDeviceForRemoteMessages,
  requestPermission,
  subscribeToTopic,
  unsubscribeFromTopic,
  setBackgroundMessageHandler,
  AuthorizationStatus,
} from '@react-native-firebase/messaging';

setBackgroundMessageHandler(getMessaging(), handler);
const authStatus = await requestPermission(getMessaging());
isDeviceRegisteredForRemoteMessages(messaging)
await registerDeviceForRemoteMessages(messaging);
const token = await getToken(messaging);
onMessage(messaging, handler);
onNotificationOpenedApp(messaging, handler);
getInitialNotification(messaging);
await subscribeToTopic(getMessaging(), topic);
await unsubscribeFromTopic(getMessaging(), topic);
authStatus === AuthorizationStatus.AUTHORIZED
```

**Pattern dùng messaging instance:**
```typescript
// Lấy instance một lần, dùng lại trong method
const messaging = getMessaging();
// ... dùng messaging nhiều lần trong cùng function
```

### Bảng mapping đầy đủ

| Namespaced (cũ) | Modular (mới) |
|---|---|
| `messaging().requestPermission()` | `requestPermission(getMessaging())` |
| `messaging().isDeviceRegisteredForRemoteMessages` | `isDeviceRegisteredForRemoteMessages(getMessaging())` |
| `messaging().registerDeviceForRemoteMessages()` | `registerDeviceForRemoteMessages(getMessaging())` |
| `messaging().getToken()` | `getToken(getMessaging())` |
| `messaging().onMessage(h)` | `onMessage(getMessaging(), h)` |
| `messaging().onNotificationOpenedApp(h)` | `onNotificationOpenedApp(getMessaging(), h)` |
| `messaging().getInitialNotification()` | `getInitialNotification(getMessaging())` |
| `messaging().setBackgroundMessageHandler(h)` | `setBackgroundMessageHandler(getMessaging(), h)` |
| `messaging().subscribeToTopic(t)` | `subscribeToTopic(getMessaging(), t)` |
| `messaging().unsubscribeFromTopic(t)` | `unsubscribeFromTopic(getMessaging(), t)` |
| `messaging.AuthorizationStatus.X` | `AuthorizationStatus.X` (import trực tiếp) |

### Checklist khi áp dụng cho project mới

- [ ] Kiểm tra version: `@react-native-firebase/messaging` >= 22 → bắt buộc dùng modular API
- [ ] Thay toàn bộ `import messaging from '@react-native-firebase/messaging'`
- [ ] Import các function riêng lẻ từ `@react-native-firebase/messaging`
- [ ] `setBackgroundMessageHandler` phải gọi ở **module level** (ngoài class/function), truyền `getMessaging()` làm arg đầu tiên
- [ ] `AuthorizationStatus` import trực tiếp, không còn là `messaging.AuthorizationStatus`

## Fix Warning `setLayoutAnimationEnabledExperimental` (New Architecture)

### Triệu chứng

```
setLayoutAnimationEnabledExperimental is currently a no-op in the New Architecture.
```

### Root Cause

React Native **New Architecture** (Fabric/JSI) không cần enable `LayoutAnimation` thủ công nữa — nó luôn được bật sẵn. Đoạn code này là legacy từ thời Old Architecture:

```typescript
// ❌ Không cần trong New Architecture
if (Platform.OS === 'android' && UIManager.setLayoutAnimationEnabledExperimental) {
  UIManager.setLayoutAnimationEnabledExperimental(true);
}
```

### Fix

Xóa hoàn toàn đoạn code trên. `LayoutAnimation` vẫn dùng bình thường, chỉ bỏ bước "enable" thủ công:

```typescript
// ✅ Vẫn dùng LayoutAnimation bình thường
LayoutAnimation.configureNext(LayoutAnimation.Presets.easeInEaseOut);
```

Sau khi xóa block `if`, dọn import — bỏ `Platform` và `UIManager` nếu không dùng ở nơi nào khác:

```typescript
// ❌ Xóa nếu không còn dùng
import { ..., Platform, UIManager } from 'react-native';

// ✅ Giữ lại nếu còn dùng LayoutAnimation
import { ..., LayoutAnimation } from 'react-native';
```

### Checklist khi áp dụng cho project mới

- [ ] Grep toàn bộ project: `grep -r "setLayoutAnimationEnabledExperimental" src/`
- [ ] Xóa toàn bộ block `if (Platform.OS === 'android' && UIManager.setLayoutAnimationEnabledExperimental)`
- [ ] Kiểm tra từng file: `Platform` và `UIManager` có còn dùng không — nếu không thì xóa khỏi import
- [ ] `LayoutAnimation.configureNext(...)` vẫn giữ nguyên, không cần thay đổi
